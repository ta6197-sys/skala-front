# 내 여행지 (myTrip)

myProfile.html과 동일한 풀페이지 스크롤 스냅(scroll-snap) 방식으로 여행지를 소개하는 페이지.
카드 그리드 대신, 한 화면에 사진 1장 + 지역명 + 한 줄 소개가 나오는 구성으로 스크롤하며 넘어간다.

> 히스토리: 카드 그리드(4개 placeholder) -> 실제 사진 6장 적용 -> myProfile과 동일한 스크롤 스냅 방식 + 영상 섹션 추가 -> 사진 전시관처럼 사진마다 성격에 맞는 프레임 방식으로 큐레이션.

## 구성 (총 7섹션, 한 섹션 = 한 화면)
1. 전북 제천 — 바닷가, 비치발리볼 (`images/trip1.jpg`) — 풍경 사진, 가로 3:2
2. 서울 — 도심 거리 (`images/trip2.jpg`) — 인물 사진, 세로 3:4, 좌측 정렬
3. 경주 — 리조트 전망 (`images/trip3.jpg`) — 풍경 사진, 가로 3:2
4. 김해 — 롯데워터파크 (`images/trip4.jpg`) — 클로즈업 셀카, 액자(매트) 프레임에 원본 그대로
5. 무주 — 스키장 (`images/trip5.jpg`) — 풍경 사진, 가로 3:2
6. 서울 — 도심 야경 (`images/trip6.jpg`) — 풍경 사진, 가로 3:2
7. 여행 영상 "전북대 축제" (`videos/trip-video.mp4`, `<video controls>`, 위쪽에 제목 텍스트)

## 사진 큐레이션 원칙 (사진 전시관처럼)
- **풍경/전경 사진**(01, 03, 05, 06): 정사각형 크롭 대신 원본에 가까운 가로 3:2 비율(`trip-photo-landscape`)로 전체 장면이 잘리지 않게 노출
- **인물 중심 사진**(02): 세로 3:4 비율(`trip-photo-portrait`)로 인물과 배경이 함께 보이도록, 필요 시 `photo-nudge-left`로 위치 미세 조정
- **클로즈업 셀카**(04): 크롭하면 부자연스러운 사진은 자르지 않고 `.photo-frame` 액자(매트+테두리)에 원본 그대로 표시
- 각 사진 크기/여백은 `css/style.css`에서 px 단위로 조정 가능하도록 주석 표기 (vw/vh 대신 고정 px 사용)
- 각 사진 섹션은 myProfile의 `.profile-section-inner--split` 구조(사진+텍스트) 재사용
- 홀/짝 섹션마다 사진이 좌우로 번갈아 배치 (`.trip-scroll` 스코프 전용 CSS, row-reverse 시 `justify-content: flex-end` 필수 — 안 붙이면 반대편에 큰 여백 생기는 버그 있었음)
- 각 섹션에 "다녀옴"/"가고 싶음" 상태 태그 표시

## 원칙
- 사이트 공통 테마(화이트×블랙×레드, 각진 모서리) 유지
- 이모지 사용하지 않음
- 외부 라이브러리 없이 순수 HTML5 `<video>` 태그로 영상 재생
- 공통 네비게이션 유지
