# 내소개 (myProfile)

SK커리어 홈페이지처럼, 스크롤을 내리면 섹션이 한 화면씩 딱딱 넘어가는 풀페이지 스크롤 스냅(scroll-snap) 방식으로 구현한다.
외부 라이브러리(fullPage.js 등) 없이 순수 CSS `scroll-snap`만으로 구현한다.

> 색상: 사이트 전체가 화이트×블랙×레드 미니멀 테마(흰 배경 + 블랙 타이포 + 레드 포인트)로 전환됨에 따라, 이 페이지도 `css/style.css`의 공통 변수(`--bg`/`--surface`/`--text`/`--accent`)를 그대로 상속받아 동일한 테마로 표시됨.

## 공통 네비게이션 레이아웃
SK Careers 사이트(로고 좌측 + 메뉴 우측) 구조를 참고하여, myProfile.html을 포함한 내부 페이지(index.html 제외 5개 페이지) 공통 네비게이션을 아래처럼 배치한다.
- **좌측 상단**: "메인" (로고 역할, index.html로 이동)
- **우측 상단**: 나머지 5개 링크를 순서대로 가로 나열 — 내소개 / 내 강의 일정 / 회원가입 / 회원가입결과 / 내 여행지

> index.html은 별도 방침(공통 네비게이션 없음, 카테고리 화면이 이동 수단)을 따르므로 이 레이아웃 대상에서 제외.

## 섹션 구성 (총 8개, 한 섹션 = 한 화면)

> 스펙을 나열하는 자기소개서 톤 대신, 사실 기반 섹션(학력/스킬/연락처)은 담백하게 남기고 나머지는 유머와 개성으로 차별화한다.

1. **인트로** — 프로필 사진 + 이름 + 웃긴 캐치프레이즈 (진지한 자기소개 아님)
2. **한 줄 소개** — 나를 요약하는 한 문장, 유머 섞어서 표현
3. **학력 / 경력** — 학교, 전공, 경력 사항을 담백하게 사실만 (진지 유지)
4. **스킬** — "잘하는 것 / 아직 배우는 중인 것"으로 솔직하게 구분 (과장 없이)
5. **저를 설명하는 말 TOP 3** — 주변 사람들이 나에게 자주 하는 말/반응 (재미 요소)
6. **저의 사용 설명서** — 제품 매뉴얼 패러디 (충전 방법, 주의사항, 모드 등으로 성향 표현)
7. **흑역사 아닌 흑역사** — 부담 없이 웃을 수 있는 실패담/에피소드 하나
8. **연락처** — 이메일, SNS 등 연락 가능한 채널 (담백하게, 마무리 섹션)

## 구현 프롬프트 (그대로 사용 가능)

```
myProfile.html을 아래 구조로 만들어줘. 외부 라이브러리는 쓰지 말고 순수 HTML + CSS만 사용해줘.

1. 공통 네비게이션(nav)
   - 좌측 상단: "메인" (index.html로 이동, 로고 역할)
   - 우측 상단: 내소개 / 내 강의 일정 / 회원가입 / 회원가입결과 / 내 여행지 링크를 가로로 나열
   - 스크롤 컨테이너 밖(최상단)에 고정

2. 전체를 감싸는 스크롤 컨테이너
   - height: 100vh, overflow-y: scroll
   - scroll-snap-type: y mandatory 적용

3. 컨테이너 안에 섹션(section)을 8개 나열
   - 각 섹션: height: 100vh, scroll-snap-align: start
   - 스크롤 시 섹션 단위로 딱딱 멈추도록(스냅) 동작
   - 섹션 순서:
     1) 인트로 (프로필 사진 + 이름 + 웃긴 캐치프레이즈)
     2) 한 줄 소개 (유머 섞어서)
     3) 학력 / 경력 (담백하게)
     4) 스킬 (잘하는 것 / 배우는 중인 것)
     5) 저를 설명하는 말 TOP 3
     6) 저의 사용 설명서 (매뉴얼 패러디)
     7) 흑역사 아닌 흑역사
     8) 연락처 (담백하게)
```

## 구현 방식 (참고용)
- `scroll-snap-type: y mandatory` — 스크롤 컨테이너에 적용
- `scroll-snap-align: start` — 각 섹션(1화면 단위)에 적용
- 각 섹션 `height: 100vh`로 화면 전체를 채움
- JS 없이 CSS만으로 스냅 효과 동작 확인 가능 (fullPage.js 같은 외부 라이브러리 불필요)

## 원칙
- 외부 라이브러리 사용 금지 (fullPage.js, GSAP 등 X) — 순수 CSS `scroll-snap`으로만 구현
- 공통 네비게이션 유지 (좌측 메인 / 우측 5개 링크 배치)
- `header`, `nav`, `section` 등 시맨틱 태그 사용

## 참고 / 트레이드오프
- 순수 CSS 방식은 부드러운 이징 애니메이션이나 좌측 점(dot) 인디케이터는 자동으로 제공되지 않음
- 그런 디테일까지 원할 경우 최소한의 바닐라 JS 추가 필요 (필수는 아님, 기본 스냅 효과만으로도 충분)
