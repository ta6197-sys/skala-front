# 회원가입 (signUp)

사용자가 제공한 참고 이미지(계정 정보/개인 프로필 정보/자기소개 3단 구성의 회원가입 폼)를 기본 구조로 하되,
전화번호 필드와 "Google 계정으로 가입" 버튼을 추가하고, 사이트의 화이트×블랙×레드 미니멀 테마(각진 모서리, 밑줄형 요소)로 재구성한다.

> 주의: Google 인증은 실제로 구현하지 않는다 (외부 라이브러리·서버 필요, frame.md의 "외부 라이브러리 금지" 원칙과 충돌).
> "Google 계정으로 가입하기" 버튼은 구조만 존재하며 클릭해도 실제 인증은 일어나지 않는다 (index.html의 검색창과 동일한 방식 — 구조만 재현).

## 폼 구성

1. **헤더**: "회원가입" 타이틀 + "SKALA-FRONT 회원가입을 환영합니다." 부제 (아이콘 없이 텍스트만, 사이트 전체 이모지 제거 방침 유지)

2. **간편 가입** (fieldset)
   - "Google 계정으로 가입하기" 버튼 (구조만, 실제 인증 없음)
   - 구분선 "또는 직접 입력"

3. **계정 정보** (fieldset)
   - 아이디 (text, required, 4~15자 영문/숫자)
   - 비밀번호 (password, required, 8자 이상)
   - 이메일 (text + `@` + select 도메인 "직접입력"/gmail.com/naver.com 등)
   - 전화번호 (tel, 신규 추가, 010-0000-0000 형식)

4. **개인 프로필 정보** (fieldset)
   - 이름 (text)
   - 생년월일 (date)
   - 성별 (radio: 남성 / 여성 / 선택안함)
   - 관심 분야 (checkbox, 중복 선택 가능): 웹 프론트엔드(Vue.js/HTML) / UI·UX 디자인 표준 / 백엔드 & 데이터베이스 / 클라우드 & 인프라
   - 가입 경로 (select)

5. **자기소개** (fieldset)
   - textarea, "나를 표현하는 한 줄 소개 또는 가입 인사" (최대 200자 안내 문구)

6. **버튼**: "동의하고 회원가입" (submit) / "다시 작성" (reset)

## 동작 방식
- 순수 바닐라 JS로 submit 이벤트를 가로채서, 입력값을 `sessionStorage`에 저장한 뒤 `signUpResult.html`로 이동
- `signUpResult.html`은 `sessionStorage`에서 값을 읽어와 화면에 표시 (서버 없이도 "회원가입 결과"를 보여주는 정적/연동 페이지 요구사항 충족)
- 백엔드 없이 프론트엔드만으로 동작하는 흐름이므로 새로고침하면 값이 사라질 수 있음 (세션 스토리지 특성, 별도 처리 없이 그대로 둠)

## 원칙
- 외부 라이브러리 사용 금지 — 순수 HTML/CSS/바닐라 JS
- 사이트 공통 테마(화이트 배경 + 블랙 텍스트 + 레드 포인트, 각진 모서리, 밑줄형 태그) 유지
- 이모지/아이콘 사용하지 않음
- 공통 네비게이션(좌측 메인 / 우측 5개 링크) 유지
